{# ============================================================================ Floating AI Chat Bubble Widget ---------------------------------------------------------------------------- Self-contained partial: HTML + scoped CSS + vanilla-JS controller. Includes nothing from outside; safe to drop into any authenticated page just before . Visibility is gated upstream by the caller via the Jinja flags `chat_widget_show` and `chat_widget_enabled` (set by the `inject_chat_widget_flags` context processor in QR_Server.py). Conventions: - All selectors are prefixed with `dims-chat-` to avoid colliding with page CSS / Bootstrap. - Theme is selected client-side from window.location.pathname: /employee-portal/* -> green gradient + POST /chat/employee everything else -> purple gradient + POST /chat/api Source links use /employee-portal/kb/ or /kb/ accordingly. - Conversation history is persisted per-context in sessionStorage so it survives page navigation within the tab. History sent to the API is truncated to the last 10 turns (matches existing behavior). ============================================================================ #}
